# 实验验证方案：第二定律的几何起源

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19537142.svg)](https://doi.org/10.5281/zenodo.19537142)

## 1. 概述

本协议为实验物理实验室提供了一份标准化的、分步骤的指南，用于检验论文中提出的离散基质框架的核心可证伪预测：
> “第二定律的几何起源：离散基质中不可约的信息差异”
> 周景松（2026）

核心预测是：地面原子钟与星载原子钟之间的高精度比对将揭示出一个无法消除的残留周期信号，其频率等于卫星的轨道频率（**f<sub>orb</sub> ≈ 1.7 × 10<sup>-4</sup> Hz**）。 该信号源于周期运动过程中，连续旋转在离散基底上发生的内在对称性破缺。

本协议既描述了**真实实验数据的分析**，也介绍了**所附模拟套件的使用**，以帮助理解和准备工作。

## 2. 理论预测摘要

### 2.1 核心预测
- **频率**：f<sub>orb</sub> = 1 / T<sub>orb</sub> ≈ **1.7 × 10<sup>-4</sup> Hz**，其中T<sub>orb</sub>为轨道周期（近地轨道约为90分钟）。
- **物理起源**：离散的基底无法完美实现卫星轨道的连续旋转对称性。这种“闭合信息差异”表现为时钟对基底基本更新速率采样的持续调制。
- **信号类型**：分数频率比对数据中的不可约残差（y(t) = (ν<sub>S</sub> - ν<sub>G</sub>) / ν<sub>G</sub>）。
- **预期振幅**：Δy ∼ τ<sub>min</sub> / τ<sub>clock</sub> ≈ 10<sup>-29</sup>。尽管这一振幅低于当前单次测量的灵敏度，但通过长期积分（数月至数年）高稳定性时钟数据，仍可实现探测。

### 2.2 为什么该测试可证伪
在经过必要的积分时间并考虑了所有已知系统误差后，若经适当校正的时钟比对数据的功率谱中未出现统计学上显著的f<sub>orb</sub>峰值，则将直接与离散基质框架的核心机制相矛盾。

## 3. 先决条件

### 3.1 数据要求（用于实际实验）
- **类型**：地面钟（G）与空间钟（S）比对得到的分数频差残差时间序列（y(t)）。
- **来源**：ACES任务、GPS/伽利略残差，或未来的光学钟任务。
- **时长**：至少6个月的连续数据。
- **质量**：数据必须经过标准的相对论校正（狭义相对论和广义相对论）。

### 3.2 软件要求
- Python 3.8 或更高版本
- 必需软件包：`numpy`、`matplotlib`、`scipy`
    ```bash
pip 安装 numpy matplotlib scipy
    ```

## 4. 验证流程

### 第一步：了解完整的模拟套件
该仓库提供了两个互补的模拟：
- **`substrate_sim.py`**：在宇宙学尺度上验证基本公理，表明熵呈单调递增趋势（S ∝ ln t）。这为不可逆性为何是根本性原理提供了理论依据。
- **`clock_comparison.py` 和 `power_spectrum.py`**：二者共同构成了**原子钟实验验证套件**。它们模拟了论文中提出的特定实验测试。

### 第二步：运行原子钟模拟（准备与理解）
在分析实际数据之前，请运行模拟，以了解预期的信号和分析流程。
1. **生成模拟残差**：
    ```bash
    python clock_comparison.py --duration 180
    ```
这会生成`residual.npy`。**请记住**：这只是一个*演示*。真实数据需要事先扣除所有已知的相对论效应和系统效应。

2. **分析模拟功率谱**：
    ```bash
    python power_spectrum.py
    ```
这将：
- 使用Welch方法计算功率谱密度。
- 在1.7e-4 Hz附近搜索峰值。
- 计算信噪比和频偏。
- 生成`power_spectrum.png/pdf`并保存数值结果。

### 第三步：准备您的实验数据（用于实际实验）
从您选择的任务中获取分数频率残差时间序列 `y(t)`。**至关重要的是，务必确保已应用所有标准相对论修正。** 将数据格式化为一维残差数组。

### 第4步：使用`power_spectrum.py`分析真实数据
修改`power_spectrum.py`中的`load_residual_data()`函数，以加载您的真实实验数据文件，而非`residual.npy`。然后运行：
```bash
python power_spectrum.py
```
第5步：统计显著性检验
要申报检测结果，峰值必须符合严格的标准：
频率匹配：在基准频率的10%范围内。
信噪比（SNR）：SNR ≥ 3。
p-value: p ≤ 0.01 from a statistical test on the PSD.
5. 验证标准
|标准|阈值|方法|
| :--- | :--- | :--- |
主要：频率匹配在f的10%范围内<次>轨道</次> |功率谱密度中的峰值检测|
|主要：统计显著性|p值≤0.01|卡方检验或等效检验|
|主要：信号持续性|在≥3个独立数据段中均存在|多段分析|
|次要：幅度一致性|与理论标度一致|与仿真对比|

成功的验证需要满足所有主要标准。
6. Reporting Guidelines
提交您的结果时，请包括：
任务详情及数据处理步骤（尤其是相对论修正）。
分析结果：检测到的频率、信噪比、p值及功率谱密度图。
您处理过的数据和分析脚本（如可能）。

数据可用性：ACES任务计划于2026至2027年执行。
分析：提供的power_spectrum.py脚本可在数小时内完成数据分析。
可行性：该测试利用了现有的空间基础设施，使其成为一项切实可行的近期基础理论探测手段。
